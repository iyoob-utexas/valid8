# Test Grid

This markdown file is the master checklist for code validation. It lists every test ID from the eight domain catalogs in `domains/code/tests/`, with tier, acceptance threshold, tooling examples, owner, and lifecycle stage.

Filter this table to the tests and lifecycle stages in scope for your project. Not every test applies to every codebase -- a script-only repository has little use for SEC-014 (SSRF), and a library with no HTTP surface has little use for the API boundary rows in the Adversarial domain. Scope deliberately and record the decision in the project's `dim_test` export.
